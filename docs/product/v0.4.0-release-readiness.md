# Academic Writing Toolkit v0.4.0 Release Readiness

`v0.4.0` is the first release that presents Academic Writing Toolkit as one coherent author-control system: evidence grounding, argument governance, bounded revision, clean-room review, and exact-artifact release.

## Release scope

- integrates the thesis-control schema v3 and revision-escalation human gates;
- adds the standalone `/revision-escalation` skill;
- adds `/argument-governance`, `/peer-review`, and `/self-review` with clean-room source boundaries;
- expands the local and Codex-plugin catalogue to 20 composable skills;
- replaces the README with an explanation-first product guide and three self-contained SVG diagrams;
- records the editable README visual source in [Figma](https://www.figma.com/design/HhaFm0uorv5oS7MsezWDN5);
- fixes the public demo's Harvard reading-note source lines so the citation audit is clean;
- moves the canonical GitHub branch from `master` to `main` and updates current public links and CI triggers;
- aligns the Codex plugin and ChatGPT App package versions at `0.4.0`.

## Branch integration record

The release integration includes every remote branch that still carried commits not already present in the previous default branch:

| Branch | Integrated content | Gate before release |
|---|---|---|
| `codex/revision-escalation-rule` | thesis-control schema v3, strict drift audit, deterministic migration, failure-atomic writes | CI success plus merged regression suite |

Older remote topic branches were already ancestors of the previous default branch. They require deletion after the new `main` ref and release tag are verified, not another merge.

| Surface | v0.4.0 state | Boundary |

## Required gates

Run from the exact release candidate ref:

```bash
make doctor
make test
make plugin-check
make chatgpt-app-check
python3 scripts/audit-public-content.py --base-dir . --json
python3 scripts/audit-citations.py --base-dir examples/demo-project --style harvard --json
python3 scripts/verify-refs.py --bib examples/demo-project/references.bib --json
python3 .claude/skills/evidence-review/scripts/check_review_package.py examples/demo-project --strict
python3 .claude/skills/release-governance/scripts/check_release_packet.py examples/demo-project --json
python3 .claude/skills/thesis-control/scripts/check_thesis_control.py examples/thesis-control-revision-escalation/approved --strict --json
xmllint --noout docs/assets/readme/*.svg
git diff --check
```

Release only if every command exits zero, the final diff review has no unresolved blocking finding, the tag points at the verified `main` commit, and `SHA256SUMS` matches the uploaded assets.
