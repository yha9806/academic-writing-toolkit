# v0.3.2 Release Readiness

`v0.3.2` is a thesis-control hardening release for users who need AI-assisted thesis edits to stay bounded, inspectable, and reversible.

## Release Scope

- Add a `/thesis-control` scaffold helper that creates draft `spine_cards.csv`, `edit_contracts.csv`, `drift_audits.csv`, and a review packet from one manuscript unit.
- Reject empty scaffold excerpts, unsafe identifiers, orphan edit contracts, uninspectable source paths, and packet paths that do not stay inside the thesis-control review root.
- Keep ChatGPT App and Codex plugin package versions aligned at `0.3.2`.
- Keep the ChatGPT App as a narrow pasted-text review surface; the full `/thesis-control` workflow remains local-agent and Codex-plugin only.
- Preserve the zero-cost update path for the already-published ChatGPT App on the existing Hugging Face Space MCP base URL.

## Required Checks

Run before tagging:

```bash
make doctor
make plugin-check
make chatgpt-app-check
python3 scripts/audit-public-content.py --base-dir . --json
make test
python3 .claude/skills/thesis-control/scripts/check_thesis_control.py <real-thesis-copy> --strict --json
git diff --check
```

For the real-thesis check, use a copied or disposable thesis project, not the author's live manuscript workspace.

## Expected Release Notes

Academic Writing Toolkit `v0.3.2` hardens `/thesis-control` with a scaffold command for draft control packets, strict path and identifier validation, empty-excerpt rejection, orphan-contract rejection, and regression tests that cover draft packet generation, unsafe identifiers, uninspectable paths, missing contract spine bindings, and relative output directories. The release keeps the ChatGPT App and Codex plugin metadata aligned while preserving the ChatGPT App as a pasted-text-only review surface.

## External Publication Boundary

The repository can tag and verify `v0.3.2`, and the Hugging Face Space can be updated from the repository app package. OpenAI dashboard review still requires the maintainer-controlled Apps Manage account, a hosted `/health` endpoint reporting version `0.3.2`, and completed organization verification before pressing Submit for Review.
