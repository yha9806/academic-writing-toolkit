# Method Limitation Boundary

This subsection justifies the use of a small qualitative corpus rather than a representative dataset. The purpose is not to estimate prevalence across students, disciplines, tools, or institutions. It is to test whether the coding framework can make revision-control failures visible within a bounded set of thesis editing logs drawn from three anonymised drafting sessions.

That boundary is part of the method. The next chapter can report pattern examples from these sessions, but those examples should not be read as population claims or as proof that a particular tool or model caused the observed problems. Claims about frequency, model capability, or institutional practice would require a larger design.

The limitation therefore clarifies what the evidence can and cannot support. The corpus is useful because it allows close inspection of evidence-boundary mistakes inside the selected sessions, while preserving the caveat that the study offers local analytic visibility rather than representative generalisation.
