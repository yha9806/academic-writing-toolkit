# Method Limitation Boundary

This subsection explains the use of a small qualitative corpus. The corpus is not intended to represent all students, disciplines, tools, or institutions. Its value is narrower: it provides enough detail to test whether the coding framework can make revision-control failures visible within anonymised thesis editing logs.

This boundary matters for the claims that follow. The next chapter can report pattern examples from the three drafting sessions, but those examples should not be read as prevalence estimates or as proof that a particular tool or model caused the observed problems. Stronger claims about frequency, model capability, or institutional practice would require a different research design.

The limitation is therefore part of the method rather than an apology for it. The edit should keep the contribution clear while preserving the caveat that this corpus supports local analytic visibility, not representative generalisation.
