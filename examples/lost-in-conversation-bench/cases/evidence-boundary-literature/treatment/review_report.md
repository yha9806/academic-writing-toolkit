# Treatment Review

## Workflow

Treatment converts the multi-turn literature-editing requirements into a source-boundary spine card and edit contract before revising the paragraph. A drift audit checks the edited paragraph against that contract.

## Metric Review

| Metric | Review |
| --- | --- |
| Spine preservation | Strong. The paragraph still motivates evidence review rather than becoming a checklist or tool evaluation. |
| Claim drift | Low and audited. The audit records `changed_claims=none`. |
| Evidence boundary | Strong. The roles of Source C and Source D remain separated and bounded. |
| Scope discipline | Strong. Checklist items and software evaluation stay outside this paragraph. |
| Author control recovery | Strong. The author can inspect which claims were protected and whether the edit respected them. |

## Control Finding

The treatment protects the source boundary as a first-class object. This makes the final paragraph easier to accept or revise because the author does not need to infer which evidence limits the model was supposed to preserve.
