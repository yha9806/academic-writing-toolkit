# Baseline A Review

## Workflow

Baseline A simulates ordinary multi-turn chat editing in which the author asks for forcefulness before later adding source-boundary caveats.

## Metric Review

| Metric | Review |
| --- | --- |
| Spine preservation | Partial. The paragraph still motivates evidence review, but it becomes a stronger claim about reliability. |
| Claim drift | High. The output claims that citation-tracking tools can verify truth and that revised thesis text cannot be treated as reliable without specific steps. |
| Evidence boundary | Weak. Source C and Source D are overstated as proof rather than background motivation. |
| Scope discipline | Weak. The paragraph starts describing checklist items that belong in the next subsection. |
| Author control recovery | Weak. The author must manually inspect each source boundary to detect the overclaim. |

## Control Finding

Baseline A shows a common literature-review drift: the edit sounds more decisive, but it asks background sources to do more work than they can support.
